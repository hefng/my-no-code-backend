## 各文件主要修改内容

### VueProjectCodegenService.java（新增）
定义 Vue 工程化代码生成的 AI 服务接口。使用 `@SystemMessage(fromResource = ...)` 加载系统提示词，返回 `TokenStream` 以支持 `onPartialThinking` / `onPartialResponse` 回调，分别获取深度推理模型的思考过程和最终答案。

### VueProjectCodegenServiceFactory.java（新增）
每次请求新建服务实例（因为 `VueProjectFileSaveTool` 持有 `appId` 状态不可复用）。注入 `reasoningStreamingChatModel`（推理模型）、Redis 对话记忆，并注册 `VueProjectFileSaveTool` 工具，让 AI 可以直接调用工具写文件。

### VueProjectFileSaveTool.java（新增）
通过 LangChain4j `@Tool` 注解暴露给 AI 的文件写入工具。AI 在生成每个文件后主动调用此工具，彻底替代"后端解析代码块"方案。包含路径穿越安全校验（禁止 `../` 逃逸项目目录），每个 `appId` 对应独立的项目目录 `{CODEGEN_DIR}/vue_project_{appId}/`。

### ReasoningStreamingChatModelConfig.java（新增）
独立配置推理流式模型 Bean（`reasoningStreamingChatModel`），开启 `returnThinking = true`，专用于 VUE_PROJECT 生成。与 langchain4j auto-config 自动创建的普通流式模型（`streamingChatModel`）并存，互不干扰。

### AiCodegenServiceFaced.java（重构）
- 统一所有流式输出格式为 `{"event": "thought"|"answer", "d": "内容片段"}`
- 新增 `generateVueProjectStream()`：通过 `Sinks.Many` 桥接 `TokenStream` 为 `Flux<String>`，分别推送 `thought` 和 `answer` 事件
- 重构 `generateCodeStream()`：HTML/MULTI_FILE 也改用 `onPartialThinking` / `onPartialResponse` 回调，支持思考过程输出
- 新增 `buildEventJson()` 工具方法统一构建事件 JSON

### AppController.java（修改）
- `chatToGenCode` 接口新增 `codegenType` 请求参数，支持前端动态指定生成类型
- 重构 SSE 事件处理逻辑：从 chunk 中解析 `event` 字段决定 SSE event 名称（`thought`/`answer`/`message`），实现向后兼容
- 对话历史收集逻辑改为只提取 `d` 字段内容，过滤掉 event 元数据

### AppServiceImpl.java（修改）
- 新增 `codegenType` 参数处理：校验合法性后覆盖应用原有类型并持久化到数据库（仅类型变化时才写库）

## 修改原因和目的

### 核心动机：支持 Vue 工程化项目生成
原有系统仅支持 HTML 单文件和 MULTI_FILE 两种代码生成模式。本次变更引入第三种模式 `VUE_PROJECT`，能够生成完整的、可运行的 Vue3 工程项目（含 `package.json`、`vite.config.js`、路由、组件等多个文件）。

### 架构决策：AI @Tool 直接写文件 vs 后端解析代码块
Vue 工程化项目涉及十几到几十个文件，如果沿用"AI 输出代码块 → 后端正则解析 → 保存文件"的方案，解析逻辑复杂且容易出错。本次采用 LangChain4j `@Tool` 方案：AI 每生成一个文件就主动调用 `VueProjectFileSaveTool.saveFile()`，文件保存的时机和内容完全由 AI 决策，后端无需解析。

### 推理模型独立配置
Vue 工程化项目生成需要更强的规划能力，因此引入支持深度思考（`returnThinking`）的推理模型，与普通流式模型分开配置，互不影响。

### SSE 事件格式统一
引入 `thought`/`answer` 事件类型，让前端能够区分"AI 思考过程"和"最终代码内容"，实现更好的用户体验（如展示"思考中..."动画）。

## 可能的影响范围

### 接口变更（需前端同步更新）
- `GET /api/app/chat/gen/code` 新增必填参数 `codegenType`，前端调用时需传入（`html` / `multi_file` / `vue_project`）
- SSE 事件格式变更：原 `message` 事件的 data 格式从 `{"d":"..."}` 变为 `{"event":"answer","d":"..."}`；新增 `thought` 事件类型。前端需适配新的事件解析逻辑

### 数据库
- 调用 `chatToGenCode` 时若传入新的 `codegenType`，会更新 `app` 表的 `codegen_type` 字段

### 基础设施依赖
- 新增 Redis 依赖（用于 Spring Session 和 LangChain4j 对话记忆存储），本地环境需确保 Redis 服务可用
- 新增推理模型 API 调用，需在配置文件中填写有效的 API Key

### 向后兼容性
- `CodeParserExecutor` 和 `CodeFileSaverExecutor` 对 `VUE_PROJECT` 返回 `null`，不影响现有 HTML/MULTI_FILE 逻辑
- `AiCodegenServiceFaced.generateAndSaveCode()`（非流式）对 `VUE_PROJECT` 抛出异常，明确不支持非流式调用
